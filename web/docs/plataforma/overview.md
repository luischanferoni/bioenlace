# Overview — Plataforma

## Qué es

Documentación transversal de **infraestructura y convenciones de desarrollo** que no pertenecen a un dominio de negocio (turnos, asistente, etc.).

## Objetivo

- Onboarding de credenciales y APIs cloud.
- Contrato para generar formularios desde docblocks de controladores API.
- Archivo histórico del plan de trabajo institucional (referencia, no roadmap de producto).

## Actores

- DevOps / backend al configurar entornos.
- Desarrolladores API que documentan parámetros de acciones CRUD/UI.
- Gestión de proyecto (plan de trabajo).

## Alcance

Incluye guías operativas y convenciones. El roadmap de producto clínico vive en [plans](../plans/README.md).
