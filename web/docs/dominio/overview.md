# Dominio transversal — Visión general

## Qué es

Documentación de **modelos y convenciones** compartidos por scheduling, clínica, organización y API (vínculo PES, relaciones ActiveRecord, parámetros de expansión en JSON).

## Objetivo

Evitar que cada módulo replique definiciones contradictorias del mismo concepto (p. ej. “profesional en efector para un servicio”).
